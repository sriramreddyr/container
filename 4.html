Running a Container

When we have the container image built using the Dockerfile ready, we are ready to create a container!!!

podman create imageName should create the container.

Podman Create Commands

Now we have the container create, to list the containers we can use
podman ps
which will get the list of the containers up and running.
podman ps -a shall be used to list all the created containers.

Podman ps Commands

Now we have the Container Image and Container Created out of the Image. Lets go ahead an run the container

podman start containerName/Image will start the Container.
Now lets run podman ps and it should list the container which we have started.

podman start will start any stopped container and this includes freshly created containers.
podman run is the combination of create and start. it creates the container and start it.
podman run -d -p 8080:8080 --name sriram-presentation imageName/ID:tag
-d is to used to detach the container and run it in background
-p is used to bind the container port to the host port.
