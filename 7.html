Volume Mounting

Now we know how to many a Container Persistent and have the required tools readily insalled once the container is created.

What about the applicaiton config files assocaited to the application running inside the container.

Lets take an example, we have an application running inside the container which needs to interact with the bankend datebase using the database credentials. Now you have a requirement to change the username/password to connect to the database.

This can be done effectively by creating a datavolumes and have the datavolume mounted to the container while creation.

In this datavolumes which can store all the app config files and can make the change to config file without effecting the state of the container.
